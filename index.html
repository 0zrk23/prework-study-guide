git

  • git works like a tree
  • "main" is the trunk
  • any feature is a branch that can be updated localy and doesn't affect the main
  • git status: command that checks the status of the branch we are on and lets you know of any changes done to the branch
  • git checkout -b "branch name": creates a branch and switches to it
  • git add -A: adds all of the changes to the stage of the current branch
  • git commit -m "description of change": adds a description of the changes made in the commit
  • git pull origin main: this command syncs the local branch to the main branch, i.e. pulls the main branch to the local one if any changes were made
  • git push origin "name of the branch you want to push": sends a push request to github so that it can be commited to the main branch
